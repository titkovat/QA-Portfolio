# API Testing with Postman

This folder contains API tests performed using **Postman** for two real projects:
- **cars.ua** — Ukrainian car marketplace (GET request)
- **GoRest** — public REST API (full CRUD cycle)

---

## 🛠 Tools & Technologies

- Postman
- REST API
- JSON
- JavaScript (Postman Scripts / assertions)

---

## 📌 Project 1 — cars.ua API

**Endpoint:** `GET https://cars.ua/api/v2/model/records?make_id=9`

**Goal:** Verify that the API returns a list of BMW car models with correct data format and status code.

---

### 📸 Screenshots

#### 01 — GET Request Setup
![GET cars.ua request](01_GET_cars_ua_request.png)

Sending a GET request to cars.ua API with query parameter `make_id=9` (BMW).
The request was discovered via Chrome DevTools → Network tab while browsing the site.

---

#### 02 — 200 OK Response
![GET cars.ua 200OK response](02_GET_cars_ua_200OK_response.png)

Server returned **200 OK** with a JSON array of BMW models.
Each model object contains: `id`, `make_id`, `name`, `cars`, `translit`.

---

#### 03 — Automated Test Scripts (3/3 passed)
![GET cars.ua scripts](03_GET_cars_ua_scripts_3tests.png)

Three automated assertions written in Postman Scripts:

```javascript
// Test 1 — Status code is 200
pm.test("Статус-код відповіді 200 OK", function () {
    pm.response.to.have.status(200);
});

// Test 2 — Response format is JSON
pm.test("Формат відповіді сервера — саме JSON", function () {
    pm.expect(pm.response.headers.get("Content-Type")).to.include("application/json");
});

// Test 3 — First model belongs to make_id 9
pm.test("Перша модель у списку належить марці з ID 9", function () {
    var jsonData = pm.response.json();
    pm.expect(jsonData.data[0].make_id).to.eql(9);
});
```
**Result: 3/3 tests passed ✅**

## 📌 Project 2 — GoRest API (CRUD)

**Base URL:** `https://gorest.co.in/public/v2/users/`

**Goal:** Perform a full CRUD cycle — Create, Update, and Delete a user — and verify each step with automated assertions.

---

### 🔄 CRUD Flow

```
POST /users → 201 Created (user_id: 8502658)
      ↓
PATCH /users/8502658 → 200 OK (status: inactive)
      ↓
DELETE /users/8502658 → 204 No Content
      ↓
GET /users/8502658 → 404 Not Found ✅
```

---

### 📸 Screenshots

#### 04 — POST Request Body
![POST create user body](04_POST_create_user_body.png)

Creating a new user with the following JSON body:

```json
{
  "name": "Anna",
  "email": "anna_test_2026_122222@gmail.com",
  "gender": "male",
  "status": "active"
}
```

---

#### 05 — POST Response: 201 Created
![POST 201 created](05_POST_create_user_201_created.png)

Server returned **201 Created** with the new user object.
A unique `id: 8502658` was automatically generated by the server.

---

#### 06 — POST Automated Assertions
![POST scripts assertions](06_POST_scripts_assertions.png)

Three automated tests written for the POST request:

```javascript
pm.test("Статус-код відповіді 201 Created", function () {
    pm.response.to.have.status(201);
});

pm.test("Користувач створений з правильним ім'ям", function () {
    var jsonData = pm.response.json();
    pm.expect(jsonData.name).to.eql("Anna");
});

pm.test("Сервер згенерував унікальний ID", function () {
    var jsonData = pm.response.json();
    pm.expect(jsonData.id).to.not.be.undefined;
});
```
**Result: 3/3 tests passed ✅**
